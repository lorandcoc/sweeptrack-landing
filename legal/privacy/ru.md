## 3. Данные, которые мы собираем (Android-приложение)

### 3.1 Данные, хранящиеся локально на вашем устройстве

Следующие данные создаются вами и хранятся **исключительно на вашем устройстве**. Мы никогда не имеем доступа к этим данным, если вы их явно не передадите.

**Данные о местоположении и сеансе:**

- GPS-координаты (широта, долгота) во время активных сеансов мониторинга
- Скорость, высота, направление и точность с датчиков устройства
- Метаданные сеанса: имя, дата, время начала/окончания, длительность, расстояние, метки
- Координаты маршрута с временными метками
- Названия мест (полученные через обратное геокодирование)

**Записи находок и открытий:**

- Тип находки (Сокровище, Золото, Монета, Реликвия, Украшение, Мусор)
- Местоположение находки (GPS-координаты на момент регистрации)
- Дополнительные метаданные: имя, заметки, глубина, оценочная стоимость, вес
- Медиа-вложения: фотографии и аудиозаписи

**Данные хранилища разрешений:**

- Названия и местоположения участков
- Имена землевладельцев и контактная информация
- Номера разрешений, даты выдачи, даты истечения срока
- Координаты границ участка (полигон)
- Цифровые подписи (формат SVG)
- Пользовательские метки и заметки

**Другие локальные данные:** Конфигурации предустановок детектора, настройки приложения (тема, единицы измерения, язык, настройки карты) и снимки погоды, связанные с сеансами.

### 3.2 Данные, обрабатываемые временно (не хранятся)

Следующие данные отправляются в сторонние API для функциональности в реальном времени и **не хранятся нами или этими службами после непосредственного запроса**:

- Приблизительные GPS-координаты, отправляемые в Open-Meteo для прогнозов погоды и данных о высоте
- Приблизительные GPS-координаты, отправляемые в OpenStreetMap/Nominatim для поиска адресов
- Приблизительные GPS-координаты, отправляемые в Overpass API для обнаружения близлежащих исторических POI
- Приблизительные GPS-координаты, отправляемые в Wikipedia API для географического поиска близлежащих статей
- Идентификаторы приливных станций, отправляемые в NOAA для прогнозов приливов
- Запросы поиска мест, отправляемые в Open-Meteo Geocoding

### 3.3 Данные, обрабатываемые сторонними службами

**Google Maps SDK:** Местоположение вашего устройства обрабатывается Google Maps для отображения карты. Подчиняется [Политике конфиденциальности Google](https://policies.google.com/privacy).

**Google Sign-In и Google Drive (необязательно):** Если вы включите резервное копирование в Google Drive, адрес электронной почты вашего Google-аккаунта сохраняется локально для идентификации аутентифицированного аккаунта. Данные резервного копирования загружаются в ваш собственный аккаунт Google Drive. Мы не имеем доступа к вашему Google Drive.

**RevenueCat (Управление подписками):** Анонимный, случайно сгенерированный идентификатор пользователя приложения используется для проверки подписки. Никакая личная информация не передается. Подчиняется [Политике конфиденциальности RevenueCat](https://www.revenuecat.com/privacy).

**Firebase Analytics и Crashlytics (Опционально — отключены по умолчанию):** Два SDK Google Firebase интегрированы в Приложение, но **оба остаются неактивными до тех пор, пока вы явно не дадите согласие** — через запрос согласия при первом запуске или в **Настройки → Конфиденциальность → Диагностика**. До получения согласия не отправляется ни одно событие и не загружается ни один отчёт о сбое. Вы можете отозвать согласие в любое время; отзыв вступает в силу со следующего события. Отладочные сборки никогда не активируют Crashlytics.

При активации:

- **Firebase Analytics** записывает восемь агрегированных имён событий с неидентифицирующими параметрами: `session_started`, `session_ended`, `find_logged`, `paywall_shown`, `premium_purchased`, `feature_gated`, `share_card_generated`, `preset_added`. Содержимое событий **никогда не включает** GPS-координаты, адреса, названия находок, фотографии, аудиозаписи, данные хранилища или любую другую персонально идентифицируемую информацию — только счётчики, длительности, расстояния, тип находки (только категорию) и идентификаторы функций.
- **Firebase Crashlytics** загружает трассировки стека сбоев вместе с моделью устройства, версией операционной системы и версией приложения, чтобы помочь нам диагностировать ошибки. Прежде чем какой-либо сбой или нефатальная ошибка передаётся в Crashlytics, Приложение **удаляет из сообщения исключения подстроки в форме координат** (например, шаблоны `lat=`/`lon=`, десятичные со знаком, содержащие три или более знака после запятой), чтобы GPS-позиции, хранящиеся в локальных переменных, не могли просочиться через отчёты об ошибках. Кадры стека (класс, метод, строка) сохраняются для группировки; значения времени выполнения — нет. Глобальный обработчик неперехваченных исключений применяет ту же очистку к фатальным сбоям, автоматически захватываемым SDK.

Обе службы подчиняются [Политике конфиденциальности Google](https://policies.google.com/privacy) и [сведениям Firebase о конфиденциальности и безопасности](https://firebase.google.com/support/privacy).

## 7. Хранение и безопасность данных

**Данные Приложения:**

- Данные сеансов и находок хранятся в базе данных Room (SQLite) в изолированном частном хранилище Приложения на вашем устройстве — защищены моделью изоляции приложений Android и шифрованием пользовательских данных устройства в состоянии покоя
- Конфиденциальные настройки используют EncryptedSharedPreferences (AES-256) Android
- Медиафайлы хранятся в частном внутреннем хранилище Приложения, недоступном для других приложений
- Резервное копирование в облако Android **отключено** (`android:allowBackup="false"`) для предотвращения непреднамеренного раскрытия данных
- Резервные копии Google Drive используют зашифрованные API Google (HTTPS/TLS) и существуют только в вашем аккаунте

Для Приложения мы не используем серверы, базы данных или облачную инфраструктуру, которые хранят ваши данные поиска.

**Данные Веб-сайта:** Электронные письма из списка ожидания хранятся в базе данных Supabase Postgres, размещенной в ЕС (Ирландия). Соединения зашифрованы (TLS). Доступ ограничен только административными целями. Исходящая электронная почта отправляется через Resend (ЕС, Ирландия) по TLS. См. Раздел 4.

## 12. Объяснение разрешений (Приложение)

- **ACCESS_FINE_LOCATION** — GPS для отслеживания маршрута во время сеансов
- **ACCESS_BACKGROUND_LOCATION** — продолжение отслеживания при выключенном экране
- **CAMERA** — съемка фотографий для регистрации находок
- **RECORD_AUDIO** — запись аудио заметок для находок
- **READ/WRITE_CALENDAR** — записывает напоминания о сроке истечения разрешений из хранилища в локальный календарь вашего устройства. Если в Android включена синхронизация с облачным календарём (например, синхронизация с Google Calendar), эти напоминания будут синхронизированы с вашим аккаунтом вместе с остальным календарём — этой синхронизацией управляют настройки Android, а не Приложение
- **INTERNET** — погода, геокодирование, приливы, карты, подписки
- **POST_NOTIFICATIONS** — уведомление об отслеживании GPS
- **VIBRATE** — оповещения о границах периметра

Вы можете отозвать любое разрешение в любое время через Настройки Android.
